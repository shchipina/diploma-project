# 💾 Стратегія резервного копіювання (Backup & Recovery Guide)

Цей документ описує процеси створення, зберігання та відновлення резервних копій для платформи SkillShare.

---

## 1. Стратегія резервного копіювання

- **Типи копій:** Використовуються **повні (Full)** резервні копії бази даних та конфігураційних файлів.
- **Частота створення:** \* **База даних:** Щоденно о 03:00 ночі (за розкладом cron).
  - **Конфігурації (`.env`):** При кожній зміні інфраструктури або оновленні.
- **Зберігання та ротація:** Копії зберігаються локально в директорії `/var/backups/skillshare/`. Зберігаються копії лише за останні 7 днів, старіші архіви автоматично видаляються (ротація).

---

## 2. Процедура резервного копіювання

### База даних (PostgreSQL)

Створення стисненого дампу бази даних:

```bash
pg_dump -U skillshare_user -d skillshare_prod -F c -f /var/backups/skillshare/db_backup_$(date +%F).dump
```

### Файли конфігурацій

Копіювання файлу `.env`:

```bash
cp /var/www/skillshare/backend/.env /var/backups/skillshare/env_backup_$(date +%F).env
```

### Логи системи

Логи керуються за допомогою вбудованого модуля PM2 (`pm2-logrotate`), який автоматично стискає старі логи та видаляє їх через 14 днів:

```bash
pm2 install pm2-logrotate
pm2 set pm2-logrotate:retain 14
pm2 set pm2-logrotate:compress true
```

---

## 3. Перевірка цілісності резервних копій

Щоб переконатися, що файл бекапу не пошкоджений, використовується команда перегляду списку (list) вмісту архіву без його фактичного розгортання:

```bash
pg_restore -l /var/backups/skillshare/db_backup_YYYY-MM-DD.dump
```

_Якщо команда виводить список таблиць без помилок — архів цілий._

---

## 4. Автоматизація процесу (Cron)

Для автоматичного створення бекапів додайте наступне завдання у `crontab -e` (виконується щодня о 03:00):

```bash
0 3 * * * pg_dump -U skillshare_user -d skillshare_prod -F c -f /var/backups/skillshare/db_backup_$(date +\%F).dump && find /var/backups/skillshare/ -type f -name "*.dump" -mtime +7 -delete
```

---

## 5. Процедура відновлення з резервних копій

### Повне відновлення системи (Disaster Recovery)

У разі критичного збою бази даних:

**Зупиніть додаток**, щоб зупинити запис нових даних:

```bash
pm2 stop skillshare-api
```

**Видаліть поточну пошкоджену базу та відновіть її з бекапу:**

```bash
pg_restore -U skillshare_user -c -d skillshare_prod /var/backups/skillshare/db_backup_YYYY-MM-DD.dump
```

**Запустіть додаток:**

```bash
pm2 start skillshare-api
```

### Тестування відновлення

Рекомендується раз на 3 місяці розгортати останній бекап на тестовому сервері (Staging), щоб гарантувати, що процедура відновлення працює коректно і дані не втрачені.
