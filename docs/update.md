# 🔄 Посібник з оновлення системи (Update & Rollback Guide)

Цей документ описує стандартизовану процедуру безпечного оновлення платформи SkillShare у Production-середовищі, а також дії у випадку критичних збоїв (Rollback).

---

## 1. Підготовка до оновлення

Перед впровадженням будь-яких змін на "живому" сервері, виконайте наступні кроки:

1. **Планування часу простою (Downtime):** Стандартне оновлення займає 2–5 хвилин. Завжди сповіщайте користувачів про технічні роботи за 24 години, якщо очікується простій більше 15 хвилин.
2. **Перевірка сумісності:** Переконайтеся, що нові зміни в коді не вимагають оновлення версії Node.js або PostgreSQL.
3. **Створення резервних копій (КРИТИЧНО):** Ніколи не оновлюйте систему без свіжого бекапу бази даних та конфігурацій:

   ```bash
   # Створення дампу бази даних
   pg_dump -U skillshare_user -d skillshare_prod -F c -f /var/backups/skillshare/db_before_update_$(date +%F).dump

   # Резервне копіювання змінних оточення
   cp /var/www/skillshare/backend/.env /var/backups/skillshare/env_before_update_$(date +%F).env
   ```

---

## 2. Процес оновлення (Deployment Flow)

Виконуйте команди послідовно в директорії бекенду (`/var/www/skillshare/backend`):

**Зупинка потрібних служб:**
Щоб уникнути конфліктів під час перезапису файлів, зупиніть поточний процес:

```bash
pm2 stop skillshare-api
```

**Розгортання нового коду:**
Завантажте останні зміни з гілки `main`:

```bash
git pull origin main
```

**Оновлення залежностей та конфігурацій:**
Якщо були змінені файли `package.json`, встановіть нові пакети (використовуйте `ci` для точної відповідності `package-lock.json`):

```bash
npm ci --only=production
```

**Міграція даних:**
Застосуйте нові схеми бази даних (якщо такі були додані розробниками):

```bash
npx prisma migrate deploy
```

**Збірка та запуск:**
Скомпілюйте новий код і запустіть сервіс:

```bash
npm run build
pm2 start skillshare-api
```

---

## 3. Перевірка після оновлення

Відразу після запуску переконайтеся, що система працює стабільно:

**Моніторинг логів:**
Перевірте останні 100 рядків логів на наявність Error або Exception:

```bash
pm2 logs skillshare-api --lines 100
```

**Тестування працездатності:**
Виконайте базовий HTTP-запит для перевірки відповіді сервера:

```bash
curl -I [http://127.0.0.1:3000/](http://127.0.0.1:3000/)
```

---

## 4. Процедура відкату (Rollback)

Якщо після оновлення система не запускається, видає критичні помилки або працює нестабільно, негайно ініціюйте процедуру відкату до попередньої робочої версії:

**Зупиніть поламаний сервіс:**

```bash
pm2 stop skillshare-api
```

**Відкотіть код до попереднього коміту:**

```bash
# Повернення на один коміт назад (або вкажіть конкретний хеш коміту)
git reset --hard HEAD~1
```

**Відновіть базу даних з бекапу (якщо міграція зламала дані):**

```bash
# Очищення поточної БД та відновлення з дампу
pg_restore -U skillshare_user -c -d skillshare_prod /var/backups/skillshare/db_before_update_YYYY-MM-DD.dump
```

**Відновіть старі залежності та перезберіть проєкт:**

```bash
npm ci --only=production
npm run build
```

**Запустіть стабільну версію:**

```bash
pm2 start skillshare-api
```
